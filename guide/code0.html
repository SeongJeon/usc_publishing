Class 네이밍 방법

최종 수정일 : 2017.00.00

1. 공통으로 들어가는 UI를 만들경우 comp-해당UI명 으로 통일한다.
ex: comp-paging , comp-tab...


2. 형태는 같으나 스타일이 다른 경우 언더바(_)로 지정한다.
ex: comp-tab_small, comp-tab_red, comp-tab_type1...


3. 클래스 네이밍 방법으로 카멜표기법은 절대 쓰지 않는다.


4. js에 관련된 클래스 작성의 경우, js-고유클래스를 만들어 작동하게 한다.


5. 전체사이트에서 쓰는 공통 페이지의 클래스 작성의 경우, #container에 tp-고유클래스를 만들어 작동하게 한다.

이미지 네이밍 방법

최종 수정일 : 2017.00.00

1. page content에 관련한 이미지의 경우, 사이트 마다의 각 대분류 폴더 안에 저장한다.

2. 공통으로 쓰이는 경우 common 폴더에 저장한다. (해당 폴더안의 스프라이트 이미지 추가를 원할 경우 요청할 것)

3. 임시로 쓰이는 이미지의 경우 사이트 마다의 각 대분류 폴더 안에 temp 폴더 생성 후, 해당 폴더에 저장해서 사용한다.

접근성

최종 수정일 : 2017.00.00

1. a 태그가 아웃링크일 경우 target="_blank" title="새창으로 열림" 해당 속성은 통일한다.

2. input, textarea, select 경우 label을 쓰지 않고(radio, checkbox 제외) title로 통일한다.
ex: 이름 입력, 내용 입력, 분류 선택 ..

3. 이미지의 경우 의미없는 이미지는 가급적 배경처리하되, 개발과 관련되어서 들어가야하는 이미지 등은 alt값을 넣어준다.

4. html작업은 논리적 순서로 마크업하며, 시멘틱하게 작업한다. (웹접근성 마크 획득 필요함)

CSS

최종 수정일 : 2017.00.00

1. css 파일의 경우, 대분류 폴더명으로 만든다.

2. 페이지마다의 style작업시에는 #container에 해당 페이지 고유 class를 부여하여 작업하는데,
css를 전체 통일해서 사용함으로 class명이 중복되지 않도록 주의한다.

3. css3의 스타일을 쓸 경우, 브라우저 하위 지원을 위해 벤더프리픽스를 같이 써준다.

HTML

최종 수정일 : 2017.00.00

1. 각 고유의 페이지명은 #container 에 클래스를 부여한다.
ex: < div id="container" class="고유 클래스명" >

2. #wrap에는 각 사이트마다의 대분류 폴더명을 클래스로 넣어준다.
ex: < div id="wrap" class="usc">

3. a / button (type=button) / button (type=submit) 의 의미에 맞게 사용한다.

반응형

최종 수정일 : 2017.00.00

1. 1024 이하부터 모바일 디바이스 적용으로 한다.

2. 1024 이하부터 font-size는 rem 단위를 적용한다. (html 폰트사이즈 10px을 고려한다.)

3. 모바일 디자인이 필요한 페이지는 요청한다.

간격가이드

최종 수정일 : 2017.00.00

margin-top : mt0 ~ mt60 (5px단위)

margin-bottom : mb0 ~ mb60 (5px단위)

margin-left : ml10 ~ ml50 (10px단위)

margin-left(마이너스값) : ml-10, ml-15, ml-20, ml-30
